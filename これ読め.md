# StarterBlog テンプレート

このリポジトリは、Astro を使った軽量なブログ用スターターテンプレートです。Fork してカスタマイズして利用できます。

---

## 前提要件

- Node.js >= 22.12.0
- Git

---

## 導入手順

1. リポジトリを fork または clone します。
2. 依存をインストールし、開発サーバーを起動します。

```bash
npm install
npm run dev
```

3. 本番ビルド:

```bash
npm run build
```

4. ビルドのプレビュー:

```bash
npm run preview
```

---

## Content Collections と frontmatter の検証

このテンプレートでは Astro の Content Collections を使って記事を管理し、Zod スキーマで frontmatter を検証します。

- スキーマ: `src/content/config.ts` に定義されています（`pubDate` の形式などを検証します）。
- 記事の配置先: `src/content/articles/`

スキーマに合わない frontmatter があると、`astro dev` や `astro build` 実行時に検証エラーが出ます。これによりメタデータの不整合を事前に防げます。

---

## 記事追加手順

1. `src/content/articles/` に新しい Markdown ファイルを作成します（例: `my-first-post.md`）。
2. ファイルの先頭に frontmatter を記述します。例:

```md
---
title: "記事タイトル"
description: "一覧に表示する短い説明"
pubDate: 2026-05-11
author: "あなたの名前"
tags:
  - Astro
  - チュートリアル
img: src/assets/thumbnail/my-thumb.png
---

本文...
```

注:
- `pubDate` は `YYYY-MM-DD` 形式である必要があります（スキーマで検証）。
- `img` は省略可能です。省略した場合は `src/assets/thumbnail/noimg.png` が使われます。

追加後、`npm run dev` で動作確認してください。検証エラーが出た場合、エラーメッセージに沿って frontmatter を修正してください。

---

## 主要ファイルの説明

- `src/content/config.ts` — Content Collection の Zod スキーマ。
- `src/content/articles/` — 記事 Markdown を配置するディレクトリ。
- `src/layouts/BlogPostLayout.astro` — 記事ページのレイアウト。
- `src/pages/article.astro` / `src/pages/article/page/[page].astro` — 記事一覧（Content Collection を利用）。
- `src/utils/articles.ts` — コレクションから記事一覧・タグを取得するユーティリティ。

---

## カスタマイズのポイント

- サイト名・説明は `src/layouts/Layout.astro` を編集してください。
- ファビコンは `public/favicon.ico` を差し替えてください。
- スタイルは `src/styles/global.css` を編集します。
