# StarterBlog テンプレート

このリポジトリは、Astro.jsを利用した記事更新の労力を最大限まで落としたブログです。

## 利用方法、使用スタック
 - Astro.js (フレームワーク)
 - Vercel (デプロイ)

## クイックスタート

このテンプレートをForkしたのち、以下のコマンドを実行します。

```bash
npm install
npm run dev
```

ビルド:

```bash
npm run build
```

プレビュー:

```bash
npm run preview
```

## 記事の追加方法（Markdown）

`src/pages/article/` に Markdown ファイルを配置し、先頭に次の frontmatter を記述します。

```md
---
layout: ../../layouts/BlogPostLayout.astro
title: "記事タイトル"
pubDate: 2026-05-11
description: "一覧に表示する短い説明"
author: "あなたの名前"
tags:
  - Astro
  - チュートリアル
img: src/assets/thumbnail/my-thumb.png
---

本文...
```

注意点：
- `pubDate` は `YYYY-MM-DD` 形式。
- `img` は省略可。省略時は `src/assets/thumbnail/noimg.png` が使われます。



## ディレクトリ構成（簡易）
- `src/pages/` — ページと記事
  - `article/` — Markdown 記事を置く
- `src/layouts/` — レイアウト（`BlogPostLayout.astro`, `Layout.astro`）
- `src/components/` — 再利用コンポーネント
- `src/assets/` — 画像など（`thumbnail/noimg.png` は残しています）
- `public/` — パブリックファイル（favicon など）

## 詳細ディレクトリ構成 (ファイルごとの説明)

- `astro.config.mjs`
  - Astro の設定ファイル。必要なインテグレーションやビルド設定を記述します。
- `tsconfig.json`
  - TypeScript の設定。IDE や型チェックに使用されます。
- `package-lock.json`
  - NPM のロックファイル（プロジェクト名は中立名 `starterblog` に変更済み）。
- `.gitignore`
  - `node_modules/`, `dist/` などコミットしないものを列挙します。

- `public/`
  - `favicon.ico`, `favicon.svg` など静的に提供するファイルを置きます。

- `src/` — サイト本体のソース
  - `src/assets/`
    - `background.svg` — レイアウトの背景や装飾に使う SVG。
    - `thumbnail/noimg.png` — サムネイル未指定時に使うプレースホルダー画像。
  - `src/components/`
    - `ArticleCard.astro` — 記事一覧や検索結果で記事カードを描画するコンポーネント。
    - `slide.astro` — スライダー等で使う軽量コンポーネント（プレースホルダとして残す場合あり）。
  - `src/layouts/`
    - `Layout.astro` — サイト全体のレイアウト（ヘッダー、ナビ、フッター）。サイト名やナビリンクはここで変更します。
    - `BlogPostLayout.astro` — 個別記事ページで使うレイアウト（メタ情報、本文のレンダリング）。
  - `src/pages/`
    - `index.astro` — ホームページ。トップ表示やピックアップ記事を配置します。
    - `about.astro` — 運営者情報などの固定ページ。
    - `policy.astro` — プライバシーや利用規約などの固定ページ。
    - `article.astro` — 記事一覧ページ（検索・タグ絞り込みを実装）。
    - `src/pages/article/`
      - `example.md` — フロントマターの書式検証用のサンプル記事。
      - `page/[page].astro` — ページネーション用のルーティングファイル。
  - `src/scripts/`
    - `article-filter.ts` — クライアント側の検索/フィルタ挙動を担うスクリプト。
    - `stack-slider.ts` — スタック表示のスライダー用スクリプト。
  - `src/styles/`
    - `global.css` — 全体のスタイル。カスタマイズ先。
  - `src/utils/`
    - `articles.ts` — Markdown を集めて記事データとして提供するユーティリティ。
    - `remark-callouts.mjs` — Markdown の呼び出し記法（callout）を変換する Remark プラグイン。

- 生成物・運用に関するファイル
  - `dist/` — ビルド出力（テンプレートとしては含めないことを推奨。現在は削除済み）。

---

上記構成をもとに、`src/layouts/Layout.astro` と `src/styles/global.css` を編集すれば目に見えるサイトの外観を変えられます。記事は `src/pages/article/` に Markdown ファイルを追加するだけで反映されます。

## カスタマイズのポイント

- サイト名・説明は `src/layouts/Layout.astro` を編集してください。
- ファビコンは `public/favicon.ico` を差し替えてください。
- スタイルは `src/styles/global.css` を編集します。
- 記事の新規タグについては、frontmatterに書くだけで自動で追加されます

## 注意事項

- 画像やコンテンツはexample.mdのみ用意してあります。自分のコンテンツを追加して利用してください。
- 変更を反映するには `npm run build` を実行して動作確認してください。
